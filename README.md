# Godje Club — Telegram-бот для бронирования компьютеров

Бот для компьютерного клуба «Godje Club». Позволяет пользователям регистрироваться, входить в систему, бронировать компьютеры, просматривать свои брони и отменять их. Администратор может управлять компьютерами, просматривать все брони и пользователей, а также вести логи действий.

---

## Что умеет

- 🔐 **Регистрация и вход**  
  Пользователь создаёт аккаунт с никнеймом, именем, телефоном и паролем (пароль хешируется через bcrypt).  
  Вход по никнейму и паролю.

- 🖥️ **Бронирование компьютера**  
  Пошаговый выбор:
  - даты начала,
  - времени начала,
  - даты окончания (до 7 дней от начала),
  - времени окончания,
  - свободного компьютера из списка доступных.  
  Проверка пересечений с другими бронями.

- 📋 **Мои бронирования**  
  Список всех активных броней пользователя с возможностью отменить любую из них.

- 💰 **Тарифы**  
  Отображение фото с ценами и акциями.

- ℹ️ **О клубе**  
  Адрес, контакты, режим работы.

- 🎧 **Оборудование**  
  Просмотр фото гарнитур и техники.

- ⚙️ **Админ-панель** (только для ADMIN_ID)  
  - просмотр всех броней,
  - просмотр всех зарегистрированных пользователей,
  - управление компьютерами (добавление, удаление, переключение активности),
  - просмотр логов действий администратора.

- ⏰ **Уведомления**  
  Автоматическое напоминание пользователю за час (настраивается) до начала брони.

## Стек

- Python 3.11+
- aiogram 3 (Telegram Bot API)
- SQLite (встроенная БД)
- bcrypt (хеширование паролей)
- Docker + docker-compose

## Структура проекта

```
godje_club_bot/
├── main.py               # Точка входа, запуск поллинга и фоновых задач
├── config.py             # Настройки из .env (токен, ADMIN_ID, пути к фото)
├── database.py           # Все запросы к SQLite (пользователи, компьютеры, брони, логи)
├── states.py             # FSM-состояния (регистрация, логин, бронирование, админка)
├── keyboards.py          # Reply и Inline клавиатуры
├── middlewares.py        # Middleware для проверки авторизации
├── handlers/             # Обработчики сообщений и callback'ов
│   ├── __init__.py       # Экспорт всех обработчиков
│   ├── common.py         # /start, тарифы, информация, оборудование, возврат в меню
│   ├── auth.py           # Регистрация и вход
│   ├── booking.py        # Бронирование, просмотр и отмена броней
│   └── admin.py          # Админ-панель (все разделы)
├── data/                 # Папка для базы данных (создаётся автоматически)
├── цены.jpg              # Фото тарифов
├── гарнитура1.jpg ...    # Фото оборудования (5 файлов)
├── requirements.txt      # Зависимости Python
├── Dockerfile
├── docker-compose.yml
└── .env.example          # Шаблон для .env
```

> Обработчики разбиты по логическим модулям, но могут быть объединены в один файл при желании.

## Запуск через Docker

1. Склонируйте репозиторий и перейдите в папку проекта.
2. Создайте файл `.env` по образцу `.env.example` и заполните:
   ```
   BOT_TOKEN=ваш_токен_бота
   ADMIN_ID=ваш_telegram_id
   ```
3. Запустите сборку и контейнер:
   ```bash
   docker compose up -d --build
   ```
4. Бот автоматически создаст базу данных и заполнит компьютеры с номерами 1–15.

## Админ-панель

Доступна только пользователю с `ADMIN_ID`, указанным в `.env`.

Команды в админ-панели:

- **Просмотр броней** – список всех бронирований с данными пользователя.
- **Просмотр пользователей** – список всех зарегистрированных пользователей.
- **Управление компьютерами**  
  - Добавить компьютер (ввод номера)  
  - Удалить компьютер (по ID, только если нет броней)  
  - Переключить активность (вкл/выкл) – неактивные компьютеры не показываются при бронировании.
- **Логи администратора** – последние 20 действий (добавление/удаление/переключение компьютеров, просмотры).

## Настройка уведомлений

В файле `config.py` задаётся переменная `NOTIFY_MINUTES` (по умолчанию 60).  
Фоновая задача проверяет каждую минуту брони, у которых время начала совпадает с текущим временем + `NOTIFY_MINUTES`, и отправляет напоминание пользователю. После отправки бронь помечается как уведомлённая, чтобы избежать дублирования.

---

## Как изменить тарифы или фото оборудования

Замените файлы `цены.jpg` и `гарнитура*.jpg` в корне проекта на свои.  
В `config.py` можно изменить список `EQUIPMENT_PHOTOS` для добавления/удаления фото.

---

## Примечания

- База данных находится в папке `data/godje_club.db` и сохраняется между перезапусками благодаря Docker-тому.
- При первом запуске автоматически создаются 15 компьютеров (номера 1–15).
- Пароли хранятся в хешированном виде (bcrypt).
- Все действия администратора логируются в таблицу `admin_logs`.
