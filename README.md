# Godje Club — Telegram-бот для бронирования компьютеров

Простой Telegram-бот для компьютерного клуба. Позволяет пользователям регистрироваться, входить в систему, бронировать компьютеры на определённое время, просматривать и отменять свои бронирования. Администратор имеет доступ к панели управления для просмотра всех броней и списка пользователей.

---

## Что умеет бот

- 🔐 **Регистрация и вход** – создание учётной записи с уникальным никнеймом и паролем, привязка к Telegram ID.
- 🖥️ **Бронирование компьютера** – выбор даты, времени начала, времени окончания (не более 12 часов), выбор свободного ПК.
- 📋 **Мои бронирования** – просмотр всех активных броней с возможностью отмены.
- 💰 **Тарифы** – отображение фото с прайсом.
- ℹ️ **О клубе** – информация об адресе, контактах, режиме работы, а также фото оборудования.
- ⚙️ **Админ-панель** – просмотр всех бронирований и списка зарегистрированных пользователей (доступна только администратору).

---

## Стек технологий

- **Python 3.11+**
- **aiogram 3** – фреймворк для Telegram Bot API
- **SQLite** – встроенная база данных (один файл)
- **python-dotenv** – управление переменными окружения

```

├── .env.example         # пример файла с переменными окружения
```

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone <url-репозитория>
cd godje_club_bot
```

### 2. Создание файла `.env`
Скопируйте `.env.example` и заполните свои данные:
```
BOT_TOKEN=ваш_токен_бота
ADMIN_ID=ваш_telegram_id
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 4. Подготовка изображений
Поместите в корень проекта:
- `цены.jpg` – фото с тарифами.
- `гарнитура1.jpg`, `гарнитура2.jpg`, …, `гарнитура5.jpg` – фотографии оборудования.

### 5. Инициализация базы данных
База данных `godje_club.db` создаётся автоматически при первом запуске. Таблицы `users`, `computers` и `bookings` будут созданы, а также 15 компьютеров (номера 1–15) добавлены в таблицу `computers`.

### 6. Запуск бота
```bash
python main.py
```

## Как пользоваться ботом

### Для обычного пользователя
1. Откройте бота и нажмите `/start`.
2. Если вы не авторизованы, бот предложит зарегистрироваться или войти.
3. После авторизации появится главное меню с кнопками:
   - **🖥️ Забронировать компьютер** – последовательный выбор даты, времени начала, времени окончания и свободного ПК.
   - **📋 Мои бронирования** – просмотр и отмена ваших броней.
   - **💰 Тарифы** – фото с ценами.
   - **ℹ️ О клубе** – информация о клубе, а также кнопка «Оборудование» для просмотра фото.

### Для администратора
- В главном меню появляется кнопка **⚙️ Админ-панель** (только если ваш Telegram ID совпадает с `ADMIN_ID` в `.env`).
- В админ-панели доступны:
  - **📋 Просмотр броней** – список всех бронирований с указанием пользователя, компьютера и времени.
  - **👥 Просмотр пользователей** – список всех зарегистрированных пользователей с их данными.

---

## Как добавить или изменить изображения

- Замените файл `цены.jpg` на своё фото с тарифами.
- Замените или дополните список `гарнитура*.jpg` – они отображаются в разделе «Оборудование».
- Пути к файлам задаются в `config.py` (переменные `TARIFFS_PHOTO_PATH` и `EQUIPMENT_PHOTOS`).

---

## Администрирование

- Единственный администратор задаётся через `ADMIN_ID` в `.env`.
- Админ-панель доступна только этому пользователю.
- В текущей версии админ может только **просматривать** данные (без возможности редактирования). При необходимости можно расширить функционал (выдача подписок, управление компьютерами и т.д.).

## Логика работы основных модулей

| Файл | Назначение |
|------|------------|
| `main.py` | Инициализация бота, диспетчера, подключение middleware и хендлеров, запуск polling. |
| `config.py` | Загрузка переменных окружения, константы путей к изображениям. |
| `database.py` | Создание таблиц, все SQL-запросы (регистрация, авторизация, проверка свободных ПК, добавление/удаление броней). |
| `keyboards.py` | Генерация reply- и inline-клавиатур для разных сценариев. |
| `middlewares.py` | Проверка авторизации для всех сообщений и callback-запросов, кроме разрешённых (вход, регистрация, /start). |
| `states.py` | Классы состояний FSM для пошаговых сценариев. |
| `utils.py` | Хеширование пароля (SHA‑256). |
| `handlers/common.py` | Команда `/start`, разделы «Тарифы», «О клубе», «Оборудование», возврат в главное меню. |
| `handlers/auth.py` | Регистрация (никнейм → имя → телефон → пароль) и вход (никнейм → пароль). |
| `handlers/booking.py` | Процесс бронирования: выбор даты, времени начала, окончания, выбор свободного ПК. Также просмотр и отмена своих броней. |
| `handlers/admin.py` | Админ-панель и вывод списков броней и пользователей. |
